### 2. Описание класса

Класс `Pair` инкапсулирует пару чисел и предоставляет интерфейс для доступа к ним через перегруженные операции.  
Операции сравнения (`==`, `!=`) реализованы как константные методы класса.  
Операция вычитания константы реализована как метод класса, модифицирующий объект.  
Операция присваивания перегружена для корректного копирования данных.  
Ввод/вывод реализован внешними дружественными функциями.

**Заголовочный файл `Pair.h`:**
```cpp
#ifndef PAIR_H
#define PAIR_H
#include <iostream>

class Pair {
private:
    int first;
    double second;
public:
    Pair();                           // без параметров
    Pair(int f, double s);            // с параметрами
    Pair(const Pair &obj);            // копирования

    void setFirst(int f);
    void setSecond(double s);
    int getFirst() const;
    double getSecond() const;

    Pair& operator=(const Pair &other);      // присваивание
    bool operator==(const Pair &other) const; // сравнение ==
    bool operator!=(const Pair &other) const; // сравнение !=
    void operator-(const double num);         // вычитание константы

    friend std::istream& operator>>(std::istream &in, Pair &pair);
    friend std::ostream& operator<<(std::ostream &out, const Pair &pair);

    ~Pair();
};

#endif
```

### 3. Определение компонентных функций

**Конструктор без параметров**
```cpp
Pair::Pair() {
    setFirst(rand() % 100);   // случайное значение 0–99
    setSecond(rand() % 100);
}
```
Инициализирует поля случайными числами через сеттеры, что позволяет в будущем добавить проверки.

**Конструктор с параметрами**
```cpp
Pair::Pair(int f, double s) {
    setFirst(f);
    setSecond(s);
}
```

**Конструктор копирования**
```cpp
Pair::Pair(const Pair &obj) {
    setFirst(obj.getFirst());
    setSecond(obj.getSecond());
}
```
Создаёт независимую копию объекта.

**Деструктор**
```cpp
Pair::~Pair() {
    cout << "Пара: " << first << " : " << second << " удалена." << endl;
}
```
Сообщает о разрушении объекта, что полезно для отслеживания времени жизни.

**Сеттеры и геттеры**
```cpp
void Pair::setFirst(int f) { first = f; }
void Pair::setSecond(double s) { second = s; }
int Pair::getFirst() const { return first; }
double Pair::getSecond() const { return second; }
```
Обеспечивают доступ к закрытым полям.

**Операция присваивания**
```cpp
Pair& Pair::operator=(const Pair &other) {
    if (this == &other) return *this;   // защита от самоприсваивания
    first = other.first;
    second = other.second;
    return *this;
}
```
Возвращает ссылку на текущий объект, что позволяет цепочечное присваивание (`a = b = c`).

**Операция сравнения `==`**
```cpp
bool Pair::operator==(const Pair &other) const {
    return (first == other.first && second == other.second);
}
```

**Операция сравнения `!=`**
```cpp
bool Pair::operator!=(const Pair &other) const {
    return !(*this == other);   // использует уже определённый operator==
}
```

**Операция вычитания константы**
```cpp
void Pair::operator-(const double num) {
    if (num - static_cast<int>(num) > 0) {   // есть дробная часть
        setSecond(getSecond() - num);        // вычесть из second
    } else {
        setFirst(getFirst() - num);          // вычесть из first
    }
}
```
Если константа вещественная (имеет ненулевую дробную часть) – уменьшается второе поле, иначе – первое.

### 4. Определение глобальных функций-операций

Операции ввода/вывода реализованы как дружественные функции, поскольку левым операндом является поток, а не объект класса.

**Оператор ввода `>>`**
```cpp
istream& operator>>(istream &in, Pair &pair) {
    cout << "first? "; in >> pair.first;
    cout << "second? "; in >> pair.second;
    return in;
}
```
Запрашивает у пользователя значения полей и возвращает ссылку на поток для возможности цепочечного ввода.

**Оператор вывода `<<`**
```cpp
ostream& operator<<(ostream &out, const Pair &pair) {
    return (out << pair.first << " : " << pair.second << endl);
    
}
```
Форматирует вывод в виде `first : second` и возвращает поток.

### 5. Функция main()

Тестовая программа последовательно проверяет все реализованные операции, а также демонстрирует работу конструкторов и деструктора.

```cpp
#include "Pair.h"
#include <iostream>
using namespace std;

int main() {
    // 1. Конструкторы
    Pair a;                        // без параметров (случайные значения)
    Pair b(10, 3.14);              // с параметрами
    Pair c(b);                     // копирования

    cout << "=== Созданные объекты ===" << endl;
    cout << "a: " << a;
    cout << "b: " << b;
    cout << "c (копия b): " << c;

    // 2. Селекторы и модификаторы
    cout << "\n=== Селекторы/модификаторы ===" << endl;
    cout << "first у b = " << b.getFirst() << ", second у b = " << b.getSecond() << endl;
    b.setFirst(42);
    b.setSecond(9.81);
    cout << "После изменений b: " << b;

    // 3. Присваивание
    a = b;
    cout << "\n=== Присваивание (a = b) ===" << endl;
    cout << "a стало: " << a;
    cout << "b осталось: " << b;

    // 4. Операции сравнения
    cout << "\n=== Сравнения ===" << endl;
    cout << "a == b ? " << (a == b ? "true" : "false") << endl;   // true
    cout << "a != c ? " << (a != c ? "true" : "false") << endl;   // true (c = 10, 3.14)

    // 5. Вычитание константы
    cout << "\n=== Вычитание константы ===" << endl;
    Pair d(20, 5.5);
    cout << "Исходная пара d: " << d;

    d - 7;                         // целое число → уменьшается first
    cout << "d - 7 (целое): " << d;

    d - 2.5;                       // дробное число → уменьшается second
    cout << "d - 2.5 (дробное): " << d;

    // 6. Ввод с клавиатуры
    Pair e;
    cin >> e;
    cout << "Вы ввели: " << e;

    cout << "\n=== Завершение программы ===" << endl;
    return 0;
}
```

### 7. Ответы на контрольные вопросы

1. **Для чего используются дружественные функции и классы?**  
   Для предоставления доступа к закрытым (private) членам класса внешним функциям или другим классам. Это расширяет интерфейс класса, не делая данные открытыми.

2. **Сформулировать правила описания и особенности дружественных функций.**  
   - Объявляются внутри класса с ключевым словом `friend`.  
   - Не являются членами класса, поэтому им не передаётся указатель `this`.  
   - Могут быть обычными функциями или методами другого класса.  
   - На них не распространяются спецификаторы доступа (место объявления в классе роли не играет).  
   - Одна функция может быть дружественной нескольким классам.

3. **Каким образом можно перегрузить унарные операции?**  
   - Как компонентную функцию класса без параметров.  
   - Как внешнюю (глобальную) функцию с одним параметром типа класса.

4. **Сколько операндов должна иметь унарная функция-операция, определяемая внутри класса?**  
   Ни одного (текущий объект является неявным операндом).

5. **Сколько операндов должна иметь унарная функция-операция, определяемая вне класса?**  
   Один операнд (объект того класса, для которого операция перегружается).

6. **Сколько операндов должна иметь бинарная функция-операция, определяемая внутри класса?**  
   Один явный операнд (второй – текущий объект, вызывающий функцию).

7. **Сколько операндов должна иметь бинарная функция-операция, определяемая вне класса?**  
   Два операнда (оба операнда передаются в параметры).

8. **Чем отличается перегрузка префиксных и постфиксных унарных операций?**  
   Префиксная форма не имеет дополнительных параметров, а постфиксная принимает фиктивный параметр типа `int` (обычно `operator++(int)`), чтобы сигнатуры различались.

9. **Каким образом можно перегрузить операцию присваивания?**  
   Перегружается как нестатическая компонентная функция класса, принимающая константную ссылку на объект того же класса и возвращающая ссылку на текущий объект (`Pair& operator=(const Pair&)`).

10. **Что должна возвращать операция присваивания?**  
    Ссылку на объект-левый операнд (`*this`), чтобы поддерживать цепочечное присваивание (`a = b = c;`).

11. **Каким образом можно перегрузить операции ввода-вывода?**  
    Исключительно как внешние дружественные функции, поскольку левым операндом являются объекты потоков (`istream` / `ostream`), которые нельзя изменить. Функции принимают ссылку на поток и объект класса, возвращают ссылку на поток.

12. **В программе описан класс `class Student { ... Student& operator++(); ... };` и определён объект `Student s;`. Выполняется операция `++s;`. Каким образом компилятор воспримет вызов функции-операции?**  
    Вызов интерпретируется как `s.operator++()`. Это компонентная функция без параметров (префиксная форма инкремента).

13. **В программе описан класс `class Student { ... friend Student& operator++(Student&); ... };` и определён объект `Student s;`. Выполняется операция `++s;`. Каким образом компилятор воспримет вызов функции-операции?**  
    Вызов интерпретируется как `operator++(s)`. Это глобальная дружественная функция с одним аргументом.

14. **В программе описан класс `class Student { ... bool operator<(Student &P); ... };` и определены объекты `Student a, b;`. Выполняется операция `cout << a < b;`. Каким образом компилятор воспримет вызов функции-операции?**  
    Сначала выполнится `cout << a`, что вызовет оператор `<<`. Затем результат сравнится с `b` с помощью `operator<`. Фактически `cout << a` возвращает ссылку на поток, а затем вызывается `a.operator<(b)`. Из-за приоритетов операций это может привести к ошибке компиляции; правильнее использовать скобки: `cout << (a < b);`.

15. **В программе описан класс `class Student { ... friend bool operator>(const Person&, Person&); ... };` и определены объекты `Student a, b;`. Выполняется операция `cout << a > b;`. Каким образом компилятор воспримет вызов функции-операции?**  
    Аналогично предыдущему случаю, сначала выполнится `cout << a`, а затем результат сравнивается с `b`. Вызовется глобальная функция `operator>(a, b)`. Здесь тоже требуется корректная расстановка скобок: `cout << (a > b);`.

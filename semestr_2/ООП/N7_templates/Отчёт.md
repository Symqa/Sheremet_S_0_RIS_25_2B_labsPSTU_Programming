### 2. Описание параметризованного класса-контейнера и пользовательского класса

**Заголовочный файл `List.h` (шаблон класса `List<T>`):**
```cpp
#ifndef LIST_H
#define LIST_H
#include <iostream>
#include <cstdlib>
using namespace std;

template <class T>
class List {
public:
    List(int s, T k);                     // конструктор с параметрами
    List(const List<T>& a);              // конструктор копирования
    ~List();                             // деструктор
    List<T>& operator=(const List<T>& a);// операция присваивания

    T& operator[](int index);            // доступ по индексу (не const)
    const T& operator[](int index) const;// доступ по индексу (const)
    int operator()() const;              // размер списка
    List<T> operator*(const T& scalar) const; // умножение на число

    template <class U>
    friend ostream& operator<<(ostream& out, const List<U>& a);
    template <class U>
    friend istream& operator>>(istream& in, List<U>& a);

private:
    int size;
    T* data;
};
#endif
```

**Заголовочный файл `Pair.h` (пользовательский класс):**

```cpp
#ifndef PAIR_H

    Pair& operator=(const Pair& p);
    Pair operator*(const Pair& p) const;  // поэлементное умножение пар

    friend ostream& operator<<(ostream& out, const Pair& p);
    friend istream& operator>>(istream& in, Pair& p);

private:
    int first;
```

### 3. Определение компонентных функций

**Методы шаблона `List<T>` (выборочно):**

*Конструктор с параметрами*
```cpp
template <class T>
List<T>::List(int s, T k) : size(s) {
    data = new T[size];
    for (int i = 0; i < size; i++)
        data[i] = k;
}
```
Создаёт динамический массив из `s` элементов, инициализированных значением `k`.

*Конструктор копирования*
```cpp
template <class T>
List<T>::List(const List<T>& a) : size(a.size) {
    data = new T[size];
    for (int i = 0; i < size; i++)
        data[i] = a.data[i];
}
```

*Деструктор*
```cpp
template <class T>
List<T>::~List() {
    delete[] data;
}
```

*Операция присваивания*
```cpp
template <class T>
List<T>& List<T>::operator=(const List<T>& a) {
    if (this == &a) return *this;
    if (data != nullptr) delete[] data;
    size = a.size;
    data = new T[size];
    for (int i = 0; i < size; i++)
        data[i] = a.data[i];
    return *this;
}
```

*Доступ по индексу*
```cpp
template <class T>
T& List<T>::operator[](int index) {
    if (index >= 0 && index < size)
        return data[index];
    else {
        cerr << "\nError! Index out of range";
        exit(1);
    }
}
```
Константная версия аналогична, но возвращает `const T&`.

*Размер списка*
```cpp
template <class T>
int List<T>::operator()() const {
    return size;
}
```

*Умножение всех элементов на скаляр*
```cpp
template <class T>
List<T> List<T>::operator*(const T& scalar) const {
    List<T> result(size, scalar); // временный объект
    for (int i = 0; i < size; i++)
        result.data[i] = data[i] * scalar;
    return result;
}
```

**Методы класса `Pair` (файл `Pair.cpp`):**

```cpp
#include "Pair.h"

Pair::Pair() : first(0), second(0.0) {}
Pair::Pair(int f, double s) : first(f), second(s) {}
Pair::Pair(int f) : first(f), second(static_cast<double>(f)) {}
Pair::Pair(const Pair& p) : first(p.first), second(p.second) {}
Pair::~Pair() {}

Pair& Pair::operator=(const Pair& p) {
    if (this != &p) {
        first = p.first;
        second = p.second;
    }
    return *this;
}

Pair Pair::operator*(const Pair& p) const {
    return Pair(first * p.first, second * p.second);
}
```

### 4. Определение глобальных функций-операций

**Операторы ввода-вывода для шаблона `List<T>`** (дружественные функции, определяются в `List.h` после объявления шаблона):
```cpp
template <class T>
ostream& operator<<(ostream& out, const List<T>& a) {
    for (int i = 0; i < a.size; i++)
        out << a.data[i] << " ";
    return out;
}

template <class T>
istream& operator>>(istream& in, List<T>& a) {
    for (int i = 0; i < a.size; i++)
        in >> a.data[i];
    return in;
}
```

**Операторы ввода-вывода для класса `Pair`:**
```cpp
ostream& operator<<(ostream& out, const Pair& p) {
    out << p.first << ":" << p.second;
    return out;
}

istream& operator>>(istream& in, Pair& p) {
    int f;
    char colon;
    double s;
    in >> f >> colon >> s;
    if (colon == ':') {
        p.first = f;
        p.second = s;
    } else {
        in.setstate(ios::failbit);
    }
    return in;
}
```

### 7. Ответы на контрольные вопросы

1. **В чем смысл использования шаблонов?**  
   Шаблоны позволяют создавать семейства функций и классов, работающих с разными типами данных, без повторного написания кода. Это обеспечивает универсальность и повторное использование алгоритмов и структур данных.

2. **Каковы синтаксис/семантика шаблонов функций?**  
   Синтаксис: `template <параметры_шаблона> заголовок_функции { тело }`. Семантика: компилятор генерирует конкретную функцию при первом вызове с соответствующим типом, подставляя параметр шаблона вместо фиктивного имени типа.

3. **Каковы синтаксис/семантика шаблонов классов?**  
   Синтаксис: `template <параметры_шаблона> class имя_класса { ... };`. Семантика: по шаблону компилятор порождает конкретный класс при объявлении объекта или указателя с указанием аргументов шаблона.

4. **Что такое параметры шаблона функции?**  
   Параметры, перечисленные после ключевого слова `template`, которые определяют типы или константы, используемые в теле функции. Например, `class T` или `int N`.

5. **Перечислите основные свойства параметров шаблона функции.**  
   - Параметры могут быть типами (`class`, `typename`) или не типовыми (целочисленные значения, указатели, ссылки).  
   - Количество параметров не ограничено.  
   - Каждый параметр должен быть известен на этапе компиляции.  
   - Параметры шаблона могут иметь значения по умолчанию.

6. **Как записывать параметр шаблона?**  
   В угловых скобках после `template`: `template <class T>` или `template <typename T>`. Для нескольких параметров: `template <class T1, class T2>`.

7. **Можно ли перегружать параметризованные функции?**  
   Да. Шаблонная функция может быть перегружена как другими шаблонными функциями (с другим количеством или типами параметров), так и обычными функциями.

8. **Перечислите основные свойства параметризованных классов.**  
   - Члены класса используют параметры шаблона.  
   - Каждый инстанцированный класс — самостоятельный тип.  
   - Статические члены существуют в единственном экземпляре для каждой инстанциации.  
   - Шаблоны классов могут быть производными и базовыми.

9. **Все ли компонентные функции параметризованного класса являются параметризованными?**  
   Нет. Компонентная функция является параметризованной, только если она сама объявлена как шаблон с дополнительными параметрами. Обычные методы класса просто используют параметры класса, не добавляя новые.

10. **Являются ли дружественные функции, описанные в параметризованном классе, параметризованными?**  
    Да, если дружественная функция использует параметры класса, она должна быть определена как шаблон с теми же параметрами (или явно специализирована), иначе возникнет ошибка линковки.

11. **Могут ли шаблоны классов содержать виртуальные компонентные функции?**  
    Да, шаблон класса может содержать виртуальные функции. Однако виртуальные функции не могут быть *шаблонами* внутри класса (т.е. шаблонная функция не может быть виртуальной).

12. **Как определяются компонентные функции параметризованных классов вне определения шаблона класса?**  
    Синтаксис: `template <class T> тип_возврата ИмяКласса<T>::имя_функции(параметры) { ... }`. Перед каждым определением должен быть заголовок шаблона.

13. **Что такое инстанцирование шаблона?**  
    Процесс порождения конкретного класса или функции из шаблона путём подстановки конкретных аргументов вместо параметров шаблона.

14. **На каком этапе происходит генерирование определения класса по шаблону?**  
    На этапе компиляции, когда компилятор встречает объявление объекта шаблонного типа с указанием конкретных аргументов (например, `List<int>`). Обычно это происходит при первом использовании в единице трансляции.

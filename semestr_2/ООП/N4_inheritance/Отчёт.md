### 2. Описание класса

Иерархия состоит из двух классов.  
**Базовый класс `Person`** инкапсулирует общие для всех людей данные (имя, возраст) и базовые операции.  
**Производный класс `Teacher`** наследует `Person` в режиме `public` и добавляет специфические для преподавателя поля (предмет, количество часов), а также методы их изменения, увеличения и уменьшения.

Отношение открытого наследования (`public`) устанавливает связь «является»: преподаватель является человеком, поэтому везде, где допустимо использование объекта `Person`, может быть использован объект `Teacher` (принцип подстановки).

**Заголовочный файл `Person.h`:**  
```cpp
#pragma once
#include <string>
#include <iostream>
using namespace std;

class Person {
protected:
    string name;
    int age;
public:
    Person();
    virtual ~Person();
    Person(string, int);
    Person(const Person&);
    string GetName() const { return name; }
    int GetAge() const { return age; }
    void SetName(string);
    void SetAge(int);
    Person& operator=(const Person&);
    friend istream& operator>>(istream& in, Person& p);
    friend ostream& operator<<(ostream& out, const Person& p);
};
```

**Заголовочный файл `Teacher.h`:**  
```cpp
#pragma once
#include "person.h"

class Teacher : public Person {
protected:
    string subject;
    int hours;
public:
    Teacher();
    ~Teacher();
    Teacher(string, int, string, int);
    Teacher(const Teacher&);
    string GetSubject() const { return subject; }
    int GetHours() const { return hours; }
    void SetSubject(string);
    void SetHours(int);
    void IncreaseHours(int delta);
    void DecreaseHours(int delta);
    Teacher& operator=(const Teacher&);
    friend istream& operator>>(istream& in, Teacher& t);
    friend ostream& operator<<(ostream& out, const Teacher& t);
};
```

### 3. Определение компонентных функций

Все методы реализованы с соблюдением правил наследования: конструктор производного класса явно вызывает конструктор базового класса, а перегруженные операторы присваивания и ввода-вывода учитывают поля обоих классов.

**Файл `Person.cpp`**  
```cpp
#include "Person.h"

Person::Person() {
    name = "";
    age = 0;
}

Person::~Person() {}

Person::Person(string N, int A) {
    name = N;
    age = A;
}

Person::Person(const Person& p) {
    name = p.name;
    age = p.age;
}

void Person::SetName(string N) { name = N; }
void Person::SetAge(int A) { age = A; }

Person& Person::operator=(const Person& p) {
    if (&p == this) return *this;
    name = p.name;
    age = p.age;
    return *this;
}

istream& operator>>(istream& in, Person& p) {
    cout << "\nName: "; in >> p.name;
    cout << "Age: "; in >> p.age;
    return in;
}

ostream& operator<<(ostream& out, const Person& p) {
    out << "\nNAME: " << p.name;
    out << "\nAGE: " << p.age;
    out << "\n";
    return out;
}
```

**Файл `Teacher.cpp`**  
```cpp
#include "Teacher.h"

Teacher::Teacher() : Person() {
    subject = "";
    hours = 0;
}

Teacher::~Teacher() {}

Teacher::Teacher(string N, int A, string S, int H) : Person(N, A) {
    subject = S;
    hours = H;
}

Teacher::Teacher(const Teacher& t) : Person(t) {
    subject = t.subject;
    hours = t.hours;
}

void Teacher::SetSubject(string S) { subject = S; }
void Teacher::SetHours(int H) { hours = H; }
void Teacher::IncreaseHours(int delta) { hours += delta; }
void Teacher::DecreaseHours(int delta) { hours -= delta; }

Teacher& Teacher::operator=(const Teacher& t) {
    if (&t == this) return *this;
    Person::operator=(t);   // копирование базовой части
    subject = t.subject;
    hours = t.hours;
    return *this;
}

istream& operator>>(istream& in, Teacher& t) {
    cout << "\nName: "; in >> t.name;
    cout << "Age: "; in >> t.age;
    cout << "Subject: "; in >> t.subject;
    cout << "Hours: "; in >> t.hours;
    return in;
}

ostream& operator<<(ostream& out, const Teacher& t) {
    out << "\nNAME: " << t.name;
    out << "\nAGE: " << t.age;
    out << "\nSUBJECT: " << t.subject;
    out << "\nHOURS: " << t.hours;
    out << "\n";
    return out;
}
```

Особенности реализации:  
- Конструктор копирования `Teacher` вызывает конструктор копирования базового класса `Person(t)`, который копирует `name` и `age`, после чего копируются локальные поля.  
- В операторе присваивания `Teacher` явно вызывается `Person::operator=(t)` для присваивания унаследованных полей.  
- Операторы ввода-вывода являются дружественными функциями и работают напрямую с защищёнными полями.

### 4. Определение глобальных функций

Для демонстрации принципа подстановки реализованы две глобальные функции:

```cpp
void f1(Person& p) {
    cout << "\n--- Inside f1 (takes Person&) ---";
    p.SetName("Ivanov");
    cout << p;
}

Person f2() {
    Teacher t("Petrov", 30, "Math", 20);
    cout << "\n--- Inside f2 (returns Person) ---";
    cout << "\nCreated Teacher: " << t;
    return t;
}
```

- **`f1`** принимает ссылку на `Person`. Благодаря принципу подстановки в неё можно передать объект `Teacher`. Внутри функции вызывается метод базового класса (`SetName`), что допустимо, так как он открыт.  
- **`f2`** возвращает объект `Person`, но внутри создаёт объект `Teacher`. При возврате происходит «срезка»: копируется только базовый подобъект `Person`, а поля `subject` и `hours` отбрасываются. Это корректно, так как возвращаемый тип – `Person`.

### 5. Функция main()

Тестовая программа последовательно проверяет создание объектов, работу перегруженных операций, модификаторы, а также принцип подстановки.

```cpp
#include <iostream>
#include "Person.h"
#include "Teacher.h"
using namespace std;

void f1(Person& p);
Person f2();

int main() {
    cout << "========== AUTOMATIC TESTS ==========\n";

    // Тестирование Person
    cout << "\n>>> Testing class Person:\n";
    Person a;
    cout << "Default Person a: " << a;
    Person b("Sidorov", 25);
    cout << "Parameterized Person b: " << b;
    Person c(b);
    cout << "Copy-constructed Person c from b: " << c;
    a = b;
    cout << "After assignment a = b, a: " << a;
    a.SetName("UpdatedName");
    a.SetAge(30);
    cout << "After setters: " << a;

    // Тестирование Teacher
    cout << "\n>>> Testing class Teacher:\n";
    Teacher t1;
    cout << "Default Teacher t1: " << t1;
    Teacher t2("Kozlov", 40, "Physics", 15);
    cout << "Parameterized Teacher t2: " << t2;
    Teacher t3(t2);
    cout << "Copy-constructed Teacher t3 from t2: " << t3;
    t1 = t2;
    cout << "After assignment t1 = t2, t1: " << t1;
    t1.IncreaseHours(5);
    cout << "After IncreaseHours(5): " << t1;
    t1.DecreaseHours(3);
    cout << "After DecreaseHours(3): " << t1;

    // Принцип подстановки
    cout << "\n>>> Testing substitution principle:\n";
    f1(t1);            // объект Teacher передаётся как Person&
    Person aa = f2();  // Teacher возвращается как Person
    cout << "After f2(), aa (Person) = " << aa;

    // Интерактивный ввод/вывод
    cout << "\n========== INTERACTIVE INPUT ==========\n";
    Person pInput;
    cout << "\nEnter data for a Person (name, age):";
    cin >> pInput;
    cout << "You entered: " << pInput;

    Teacher tInput;
    cout << "\nEnter data for a Teacher (name, age, subject, hours):";
    cin >> tInput;
    cout << "You entered: " << tInput;

    cout << "\n========== ALL TESTS COMPLETED ==========\n";
    return 0;
}
```

### 7. Ответы на контрольные вопросы

1. **Для чего используется механизм наследования?**  
   Наследование позволяет создавать иерархии классов, повторно использовать код и интерфейсы базовых классов, дополнять и изменять их поведение в производных классах.

2. **Каким образом наследуются компоненты класса, описанные со спецификатором `public`?**  
   `public`-компоненты остаются `public` в производном классе при открытом наследовании и доступны всем функциям, имеющим доступ к производному классу.

3. **Каким образом наследуются компоненты класса, описанные со спецификатором `private`?**  
   `private`-компоненты не наследуются, они недоступны в производном классе. Доступ к ним возможен только через открытые или защищённые методы базового класса.

4. **Каким образом наследуются компоненты класса, описанные со спецификатором `protected`?**  
   `protected`-компоненты при открытом наследовании остаются `protected` в производном классе и могут использоваться методами производного класса, но не извне.

5. **Каким образом описывается производный класс?**  
   Синтаксис: `class ИмяПроизводного : спецификатор_доступа ИмяБазового { ... };`. Спецификатор доступа определяет тип наследования (`public`, `protected`, `private`).

6. **Наследуются ли конструкторы?**  
   Нет, конструкторы не наследуются. Производный класс обязан явно вызывать конструктор базового класса в списке инициализации.

7. **Наследуются ли деструкторы?**  
   Деструкторы не наследуются в обычном смысле. Каждый класс должен иметь собственный деструктор, который автоматически вызывается при уничтожении объекта. Однако если деструктор определён как виртуальный в базовом классе, он фактически наследуется через механизм виртуальных функций.

8. **В каком порядке конструируются объекты производных классов?**  
   Снизу вверх по иерархии: сначала вызывается конструктор базового класса, затем конструкторы объектов-членов (если есть), и в последнюю очередь – конструктор производного класса.

9. **В каком порядке уничтожаются объекты производных классов?**  
   В порядке, обратном конструированию: сначала деструктор производного класса, затем деструкторы объектов-членов, затем деструктор базового класса.

10. **Что представляют собой виртуальные функции и механизм позднего связывания?**  
    Виртуальная функция – функция-член, объявленная с ключевым словом `virtual`. Позднее связывание (динамическое) означает, что решение о том, какая версия виртуальной функции будет вызвана, принимается во время выполнения на основе фактического типа объекта, на который указывает указатель или ссылка, а не на основе типа указателя/ссылки.

11. **Могут ли быть виртуальными конструкторы? Деструкторы?**  
    Конструкторы не могут быть виртуальными, так как на момент их вызова объект ещё не сконструирован, и механизм виртуальности не работает. Деструкторы могут и должны быть виртуальными, если класс предназначен для наследования и в нём есть виртуальные функции, чтобы обеспечить корректное удаление объектов производных классов через указатель базового класса.

12. **Наследуется ли спецификатор `virtual`?**  
    Да, виртуальность наследуется. Если функция объявлена виртуальной в базовом классе, её переопределение в производном классе автоматически остаётся виртуальным, даже если не указано слово `virtual`.

13. **Какое отношение устанавливает между классами открытое наследование?**  
    Отношение «является» (is-a). Объект производного класса может использоваться везде, где ожидается объект базового класса, с сохранением корректности поведения.

14. **Какое отношение устанавливает между классами закрытое наследование?**  
    Отношение «реализован посредством» (has-a / implemented-in-terms-of). Закрытое наследование наследует реализацию, но не интерфейс; публичные члены базового класса становятся приватными в производном, и объект производного класса не может быть подставлен вместо базового.

15. **В чем заключается принцип подстановки?**  
    Принцип подстановки (Лисков) утверждает, что объекты производного класса должны быть способны использоваться вместо объектов базового класса без нарушения корректности программы. В C++ это реализуется через открытое наследование: указатель или ссылка на базовый класс могут указывать на объект производного класса, при этом вызываются методы, определяемые динамическим типом объекта.

16. **Имеется иерархия классов:**

    - `name` (открытое поле `Student`),

    - `stage` (защищённое поле `Teacher`),

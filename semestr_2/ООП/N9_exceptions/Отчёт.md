### 2. Словесное описание исключительных ситуаций

В процессе работы со списком могут возникать следующие нештатные ситуации:

- **Превышение максимально допустимого размера** (MAX_SIZE = 30) при создании списка или при добавлении элементов.  
- **Некорректный индекс** в операции `[]`:  
  - индекс меньше 0;  
  - индекс больше или равен текущему размеру списка.  
- **Попытка перемещения курсора в пустом списке** (оператор `- n` при `size == 0`).  
- **Выход курсора за допустимые границы** при выполнении `- n`: результирующая позиция `< 0` или `>= size`.  

Все эти ситуации обрабатываются генерацией исключений в соответствующих методах класса.

### 3. Вариант реализации 1 (стандартный тип данных)

Информация об исключении передаётся целочисленным кодом.

#### 3.1. Заголовочный файл `List.h`

```cpp
#pragma once
#include <iostream>
using namespace std;

const int MAX_SIZE = 30;

class List {
    struct Node {
        int data;
        Node* next;
        Node(int d = 0, Node* n = nullptr) : data(d), next(n) {}
    };
    Node* head;
    int sz;
    mutable int current;      // позиция курсора (-1, если список пуст)
    Node* getNode(int i) const;
public:
    List();
    List(int n, int* arr = nullptr);
    List(const List&);
    ~List();
    List& operator=(const List&);

    int& operator[](int i);          // доступ по индексу
    int  operator()() const;         // размер списка
    List operator*(int factor) const; // умножение на число
    int& operator-(int n);           // переход влево на n

    friend ostream& operator<<(ostream& out, const List& l);
    friend istream& operator>>(istream& in, List& l);
};
```

#### 3.2. Определение компонентных функций (`List.cpp`)

```cpp
#include "List.h"
using namespace std;

List::List() : head(nullptr), sz(0), current(-1) {}

List::List(int n, int* arr) : head(nullptr), sz(0), current(-1) {
    if (n > MAX_SIZE) throw 1;          // код 1 – превышение MAX_SIZE
    if (n < 0) throw 1;
    for (int i = n - 1; i >= 0; --i) {
        head = new Node(arr ? arr[i] : 0, head);
        ++sz;
    }
    if (sz > 0) current = 0;
}

List::List(const List& l) : head(nullptr), sz(0), current(l.current) {
    Node** p = &head;
    for (Node* cur = l.head; cur; cur = cur->next) {
        *p = new Node(cur->data);
        p = &((*p)->next);
        ++sz;
    }
}

List::~List() {
    while (head) {
        Node* t = head;
        head = head->next;
        delete t;
    }
}

List& List::operator=(const List& l) {
    if (this == &l) return *this;
    while (head) { Node* t = head; head = head->next; delete t; }
    head = nullptr; sz = 0;
    Node** p = &head;
    for (Node* cur = l.head; cur; cur = cur->next) {
        *p = new Node(cur->data);
        p = &((*p)->next);
        ++sz;
    }
    current = l.current;
    return *this;
}

List::Node* List::getNode(int i) const {
    if (i < 0 || i >= sz) return nullptr;
    Node* cur = head;
    for (int k = 0; k < i; ++k) cur = cur->next;
    return cur;
}

int& List::operator[](int i) {
    if (i < 0) throw 2;                 // индекс < 0
    if (i >= sz) throw 3;               // индекс >= размера
    return getNode(i)->data;
}

int List::operator()() const {
    return sz;
}

List List::operator*(int factor) const {
    List res(*this);
    for (Node* cur = res.head; cur; cur = cur->next)
        cur->data *= factor;
    return res;
}

int& List::operator-(int n) {
    if (sz == 0) throw 4;               // пустой список
    int newCur = current - n;
    if (newCur < 0 || newCur >= sz) throw 5; // выход за границы
    current = newCur;
    return getNode(current)->data;
}

ostream& operator<<(ostream& out, const List& l) {
    if (l.sz == 0) out << "Empty";
    else for (Node* cur = l.head; cur; cur = cur->next) out << cur->data << " ";
    out << endl;
    return out;
}

istream& operator>>(istream& in, List& l) {
    while (l.head) { Node* t = l.head; l.head = l.head->next; delete t; }
    l.sz = 0; l.current = -1;
    cout << "Enter number of elements: ";
    int n; in >> n;
    if (n > MAX_SIZE) throw 1;
    cout << "Enter " << n << " elements:\n";
    List::Node** p = &l.head;
    for (int i = 0; i < n; ++i) {
        int val;
        cout << ">"; in >> val;
        *p = new List::Node(val);
        p = &((*p)->next);
        ++l.sz;
    }
    if (l.sz > 0) l.current = 0;
    return in;
}
```

### 4. Вариант реализации 2 (пользовательский класс `error`)

Для передачи информации об исключении создаётся класс `error`, содержащий строковое сообщение.

#### 4.1. Класс `error` (`error.h`)

```cpp
#pragma once
#include <string>
#include <iostream>
using namespace std;

class error {
    string str;
public:
    error(string s) : str(s) {}
    void what() const { cout << str << endl; }
};
```

#### 4.2. Модифицированные компонентные функции (`List.cpp`)

Все `throw int` заменены на `throw error("сообщение")`. Показаны только изменённые фрагменты.

```cpp
#include "List.h"
#include "error.h"

List::List(int n, int* arr) : head(nullptr), sz(0), current(-1) {
    if (n > MAX_SIZE) throw error("Size exceeds MAX_SIZE");
    if (n < 0) throw error("Negative size");
    // ... остальное без изменений
}

int& List::operator[](int i) {
    if (i < 0) throw error("Index < 0");
    if (i >= sz) throw error("Index >= size");
    return getNode(i)->data;
}

int& List::operator-(int n) {
    if (sz == 0) throw error("Cannot move in empty list");
    int newCur = current - n;
    if (newCur < 0 || newCur >= sz) throw error("Cursor out of bounds");
    current = newCur;
    return getNode(current)->data;
}

// операторы << и >> – аналогично, при превышении MAX_SIZE генерируется
// throw error("Size exceeds MAX_SIZE");
```

Остальные методы (копирования, деструктор, присваивание) не меняются.

### 6. Ответы на контрольные вопросы

1. **Что представляет собой исключение в C++?**  
   Исключение – это объект, который генерируется при возникновении непредвиденной или аварийной ситуации. Он позволяет передавать управление из места обнаружения ошибки в место её обработки.

2. **На какие части исключения позволяют разделить вычислительный процесс? Достоинства такого подхода?**  
   - Часть **обнаружения** аварийной ситуации – генерирует исключение, но не обрабатывает его.  
   - Часть **обработки** – перехватывает исключение и реагирует на него.  
   Достоинства: удобство в многомодульных программах; отсутствие необходимости возвращать коды ошибок через значения функций.

3. **Какой оператор используется для генерации исключительной ситуации?**  
   `throw <выражение>;`

4. **Что представляет собой контролируемый блок? Для чего он нужен?**  
   Блок `try { ... }` определяет участок кода, в котором могут возникнуть исключения. Переменные, объявленные в нём, локальны.

5. **Что представляет собой секция-ловушка? Для чего она нужна?**  
   Блок `catch(тип исключения) { ... }` перехватывает и обрабатывает исключение соответствующего типа. Может быть несколько секций-ловушек для разных типов.

6. **Какие формы может иметь спецификация исключения в секции-ловушке?**  
   - `catch(тип)` — перехват конкретного типа.  
   - `catch(...)` — перехват любого исключения.  
   Используются для обработки ожидаемых ошибок или для универсальной очистки ресурсов.

7. **Какой стандартный класс можно использовать для создания собственной иерархии исключений?**  
   Класс `std::exception`, объявленный в `<exception>`. Он содержит виртуальный метод `what()`, возвращающий C-строку с описанием ошибки.

8. **Каким образом можно создать собственную иерархию исключений?**  
   Создать базовый класс исключения (возможно, унаследованный от `std::exception`), затем определить производные классы, переопределяя метод `what()` и/или добавляя новые члены.

9. **Если спецификация исключений имеет вид `void f1() throw(int,double);` то какие исключения может порождать функция `f1()`?**  
   Функция может генерировать исключения типов `int` и `double`. (В современном C++ такая спецификация считается устаревшей, вместо неё используется `noexcept`).

10. **Если спецификация исключений имеет вид `void f1() throw();` то какие исключения может порождать функция `f1()`?**  
    Функция не должна генерировать никаких исключений. В современном C++ эквивалент – `void f1() noexcept`.

11. **В какой части программы может генерироваться исключение?**  
    Исключение может быть сгенерировано в любом месте программы, где есть оператор `throw`: в функциях, методах, конструкторах, деструкторах (хотя генерация исключений из деструкторов крайне не рекомендуется), в блоках `try`, а также внутри стандартных операций вроде `new` (при нехватке памяти).

12. **Написать функцию, которая вычисляет площадь треугольника по трём сторонам (формула Герона). Реализовать в четырёх вариантах.**  

    *Без спецификации исключений:*  
    ```cpp
    double area(double a, double b, double c) {
        if (a+b <= c || a+c <= b || b+c <= a)
            throw "Invalid triangle";
        double p = (a+b+c)/2;
        return sqrt(p*(p-a)*(p-b)*(p-c));
    }
    ```

    *Со спецификацией `throw()`:*  
    ```cpp
    double area(double a, double b, double c) throw() {
        if (a+b <= c || a+c <= b || b+c <= a) return -1;
        double p = (a+b+c)/2;
        return sqrt(p*(p-a)*(p-b)*(p-c));
    }
    ```

    *С конкретной спецификацией со стандартным исключением:*  
    ```cpp
    #include <stdexcept>
    double area(double a, double b, double c) throw(std::invalid_argument) {
        if (a+b <= c || a+c <= b || b+c <= a)
            throw std::invalid_argument("Invalid triangle sides");
        double p = (a+b+c)/2;
        return sqrt(p*(p-a)*(p-b)*(p-c));
    }
    ```

    *С собственным реализованным исключением:*  
    ```cpp
    class TriangleError : public std::exception {
    public:
        const char* what() const noexcept override {
            return "Invalid triangle sides";
        }
    };
    double area(double a, double b, double c) {
        if (a+b <= c || a+c <= b || b+c <= a) throw TriangleError();
        double p = (a+b+c)/2;
        return sqrt(p*(p-a)*(p-b)*(p-c));
    }
```

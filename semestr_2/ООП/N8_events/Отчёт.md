### 2. Описание класса-контейнера

Класс-контейнер `Tree` представляет собой группу, хранящую указатели на объекты базового абстрактного класса `Object`. Физически `Tree` реализован как динамический массив указателей `Object** beg`. Он предоставляет методы добавления, удаления, просмотра, доступа по индексу и обработки событий. Группа не просто хранит объекты, но и управляет их жизненным циклом (деструктор освобождает память). Это удовлетворяет понятию группы как объекта, обладающего собственными свойствами, не вытекающими из свойств элементов.

**Заголовочный файл `Tree.h` (фрагмент):**
```cpp
class Tree {
public:
    Tree();
    virtual ~Tree();
    Tree(int n);                 // конструктор с параметром – размером
    void Add();                  // добавить элемент (интерактивно)
    void Del();                  // удалить последний добавленный элемент
    void Show();                 // просмотреть все элементы
    int  operator()();           // вернуть текущее количество элементов
    virtual void HandleEvent(TEvent& event); // обработчик событий группы
protected:
    Object** beg;                // массив указателей на объекты
    int size;                    // максимальный размер
    int cur;                     // текущее количество элементов
};
```

### 3. Определение компонентных функций

#### 3.1. Класс `Object` (абстрактный базовый)

Класс задаёт интерфейс для всех элементов группы.

```cpp
class Object {
public:
    Object();
    virtual ~Object();
    virtual void Show() = 0;
    virtual void Input() = 0;
    virtual void HandleEvent(const TEvent& e) = 0;  // для обработки событий объектом
};
```

Реализация пуста, так как все методы чисто виртуальные.

#### 3.2. Класс `Print` (печатное издание)

Наследует `Object`, содержит поля `name`, `author`.  
Реализует конструкторы (без параметров, с параметрами, копирования), деструктор, оператор присваивания, селекторы/модификаторы, ввод/вывод и обработчик событий.

Пример конструктора с параметрами и оператора присваивания:
```cpp
Print::Print(string n, string a) : name(n), author(a) {}
Print& Print::operator=(const Print& p) {
    if (&p == this) return *this;
    name = p.name;
    author = p.author;
    return *this;
}
```

Метод `HandleEvent` для команды `cmGet` выводит название:
```cpp
void Print::HandleEvent(const TEvent& e) {
    if (e.what == evMessage) {
        if (e.command == cmGet) {
            cout << "Name = " << name << endl;
        }
    }
}
```

#### 3.4. Класс `Tree` (группа-дерево)

**Конструктор с параметром** выделяет память под массив указателей:
```cpp
Tree::Tree(int n) {
    size = n;
    beg = new Object*[size];
    cur = 0;
}
```

**Деструктор** освобождает память всех хранимых объектов и массива:
```cpp
Tree::~Tree() {
    for (int i = 0; i < cur; ++i)
        delete beg[i];
    delete[] beg;
}
```

**Добавление элемента** – пользователь выбирает тип (`Print` или `Book`), создаётся объект, вызывается `Input()`, и указатель помещается в массив:
```cpp
void Tree::Add() {
    Object* p = 0;
    cout << "1. Print\n2. Book\n";
    int choice; cin >> choice;
    if (choice == 1) {
        Print* a = new Print; a->Input(); p = a;
    } else if (choice == 2) {
        Book* b = new Book; b->Input(); p = b;
    }
    if (cur < size) { beg[cur] = p; ++cur; }
}
```

**Удаление** – уменьшает счётчик (без освобождения памяти, этого требует задание: «память не освобождается!», оставлено как в примере):
```cpp
void Tree::Del() {
    if (cur > 0) --cur;
}
```

**Просмотр** – перебор указателей через `Object** p` и вызов виртуального `Show()`:
```cpp
void Tree::Show() {
    if (cur == 0) cout << "Empty" << endl;
    Object** p = beg;
    for (int i = 0; i < cur; ++i) {
        (*p)->Show();
        ++p;
    }
}
```

**Обработчик событий группы** – для команды `cmGet` извлекает индекс из `event.a` и вызывает обработчик конкретного элемента:
```cpp
void Tree::HandleEvent(TEvent& event) {
    if (event.what == evMessage) {
        if (event.command == cmGet) {
            int index = event.a;
            if (index >= 0 && index < cur) {
                beg[index]->HandleEvent(event);
            }
            event.what = evNothing; // очистка после обработки
        }
    }
}
```

#### 3.5. Структура события и константы команд

Файл `Event.h` определяет тип события и коды команд:
```cpp
const int evNothing = 0;
const int evMessage = 100;
const int cmAdd = 1, cmDel = 2, cmGet = 5, cmShow = 4, cmMake = 6, cmQuit = 101;

struct TEvent {
    int what;
    union {
        int command;
        struct {
            int message;
            int a;      // параметр команды (например, индекс)
        };
    };
};
```

#### 3.6. Класс `Dialog` – управляющее ядро

`Dialog` наследует `Tree` и реализует главный цикл обработки событий, функции получения события, его обработки и очистки.

**Получение события (GetEvent):**  
Считывает строку, первый символ определяет команду. Если команда имеет параметр (например, `m2` или `z0`), остаток строки преобразуется в целое и помещается в `event.a`.  
```cpp
void Dialog::GetEvent(TEvent& event) {
    string OpInt = "+-szqm";
    string s; cout << "> "; cin >> s;
    // ... определение команды ...
    if (code == 'm' || code == 'z') {
        string param = s.substr(1);
        event.a = atoi(param.c_str());
    }
}
```

**Главный цикл (Execute):**  
```cpp
int Dialog::Execute() {
    TEvent event;
    do {
        EndState = 0;
        GetEvent(event);
        HandleEvent(event);
    } while (!Valid());
    return EndState;
}
```

**Обработчик событий (HandleEvent):**  
Обрабатывает команды `cmMake`, `cmAdd`, `cmDel`, `cmShow`, `cmQuit`. Если команда не распознана, передаёт событие базовому обработчику `Tree::HandleEvent` для обработки команд уровня объектов (таких как `cmGet`).  
```cpp
void Dialog::HandleEvent(TEvent& event) {
    if (event.what == evMessage) {
        switch (event.command) {
            case cmMake: /* выделение памяти */ ClearEvent(event); break;
            case cmAdd: Add(); ClearEvent(event); break;
            case cmDel: Del(); ClearEvent(event); break;
            case cmShow: Show(); ClearEvent(event); break;
            case cmQuit: EndExec(); ClearEvent(event); break;
            default: Tree::HandleEvent(event);
        }
    }
}
```

**Очистка события (ClearEvent):**  
```cpp
void Dialog::ClearEvent(TEvent& event) {
    event.what = evNothing;
}
```

**Завершение работы:**  
Метод `EndExec()` устанавливает `EndState = 1`. Метод `Valid()` возвращает `true`, если `EndState != 0`, что останавливает главный цикл.

### 4. Описание итератора

Явный класс-итератор в программе не реализован. Доступ к элементам контейнера осуществляется через указатель `Object** beg`. В методе `Show()` и при обработке события `cmGet` используется простейший перебор: объявляется вспомогательный указатель `Object** p = beg`, который инкрементируется в цикле. Такой подход аналогичен итератору произвольного доступа.

### 7. Ответы на контрольные вопросы

1. **Класс-группа** – объект, в который включены другие объекты (элементы группы), причём группа обладает собственными свойствами, не сводящимися к свойствам элементов. Примеры: окно с элементами управления, факультет со студентами, программа как группа всех своих интерактивных объектов.

2. Пример описания класса-группы Список (`List`) может выглядеть так:
   ```cpp
   class List {
       Object** data;
       int size, count;
   public:
       List();
       List(int n);
       ~List();
       void Add(Object* obj);
       void Remove(int index);
       void Show();
   };
   ```

3. Конструкторы для `List`:  
   - Без параметров: `List() : size(10), count(0) { data = new Object*[size]; }`  
   - С параметром: `List(int n) : size(n), count(0) { data = new Object*[size]; }`  
   - Копирования: `List(const List& other)` с глубоким копированием массива указателей.

4. Деструктор для `List`:
   ```cpp
   List::~List() {
       for (int i = 0; i < count; ++i) delete data[i];
       delete[] data;
   }
   ```

5. Метод просмотра для `List`:
   ```cpp
   void List::Show() {
       for (int i = 0; i < count; ++i)
           data[i]->Show();
   }
   ```

6. Группа даёт иерархию объектов (целое/часть), построенную на основе агрегации.

7. Абстрактный класс во главе иерархии элементов группы позволяет унифицировать интерфейс всех элементов (чисто виртуальные функции `Show`, `Input`, `HandleEvent`). Группа работает с указателями на абстрактный класс, что обеспечивает полиморфное поведение.

8. **Событие** – пакет информации, которым обмениваются объекты в ответ на действия пользователя или системы. События управляют работой объектов, инициируя создание, модификацию или уничтожение объектов.

9. Событие-сообщение должно содержать: код класса сообщения (тип события), адрес объекта-получателя (может отсутствовать) и информационное поле.

10. Пример структуры:
    ```cpp
    struct TEvent {
        int what;
        union {
            int command;
            struct { int message; int a; };
        };
    };
    ```

11. Полю `what` присваиваются значения:  
    - `evNothing` (0) – пустое событие, означающее, что событие обработано или ничего делать не нужно.  
    - `evMessage` (100) – событие-сообщение от объекта, требующее обработки.

12. Полю `command` присваивается числовой код команды (например, `cmAdd = 1`, `cmDel = 2` и т.д.) только когда `what == evMessage`. Этот код определяет, какое действие должно быть выполнено.

13. Поля `a` и `message` используются для передачи параметров команды. Например, в нашем проекте `a` хранит индекс элемента для команды `z`, или размер группы для команды `m`.

14. Для организации обработки сообщений необходимы методы:  
   - `GetEvent` – формирование события (ввод от пользователя).  
   - `Execute` – главный цикл обработки.  
   - `HandleEvent` – обработчик события в каждом классе.  
   - `ClearEvent` – очистка события после обработки.

15. Главный цикл обработки:
    ```cpp
    do {
        endState = 0;
        GetEvent(event);
        HandleEvent(event);
    } while (!Valid());
    ```

16. `ClearEvent()` присваивает полю `event.what` значение `evNothing`, помечая событие как обработанное, чтобы оно не передавалось дальше по цепочке.

17. `HandleEvent()` распознаёт событие по коду команды и выполняет соответствующие действия (добавление, удаление, вывод, завершение). В группе после собственной обработки передаёт событие элементам, если оно не обработано.

18. `GetEvent()` формирует событие на основе внешних воздействий. В консольной программе он считывает строку, разбирает её и заполняет поля структуры `TEvent`.

19. Поле `EndState` используется как флаг завершения работы программы. Когда оно устанавливается в ненулевое значение, метод `Valid()` возвращает `true`, и главный цикл прекращается. Это поле принадлежит классу `Dialog` (или главной группе-приложению).

20. Функция `Valid()` проверяет значение `EndState`. Если оно не равно нулю, возвращается признак завершения. Это гарантирует, что все активные элементы завершат работу перед выходом из программы.

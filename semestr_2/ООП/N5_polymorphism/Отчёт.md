### 2. Описание классов

Проект состоит из шести файлов:  
`Object.h`, `Object.cpp` – абстрактный базовый класс.  
`Person.h`, `Person.cpp` – класс, наследующий `Object`.  
`Teacher.h`, `Teacher.cpp` – производный от `Person` класс.  
`Vector.h`, `Vector.cpp` – контейнер для полиморфных объектов.  
`main.cpp` – тестовая программа.

#### 2.1. Абстрактный класс `Object`
Содержит чисто виртуальный метод `Show() = 0`, что делает класс абстрактным. Конструктор и виртуальный деструктор обеспечивают корректное удаление производных объектов через указатель базового класса.

```cpp
class Object {
public:
    Object(void);
    virtual ~Object(void);
    virtual void Show() = 0;
};
```

#### 2.2. Класс `Person` (наследует `Object`)
Хранит имя и возраст. Переопределяет виртуальный метод `Show()`. Предоставляет сеттеры/геттеры, конструкторы, оператор присваивания, дружественные операторы ввода/вывода.

```cpp
class Person : public Object {
    string name;
    int age;
public:
    Person();
    virtual ~Person();
    void Show();
    Person(string, int);
    Person(const Person&);
    string GetName();
    int GetAge();
    void SetName(string);
    void SetAge(int);
    Person& operator=(const Person&);
    friend istream& operator>>(istream&, Person&);
    friend ostream& operator<<(ostream&, const Person&);
};
```

#### 2.3. Класс `Teacher` (наследует `Person`)
Добавляет поля `subject` (предмет) и `hours` (часы). Переопределяет `Show()`, дополняя вывод родительского класса. Определены методы `IncreaseHours(int)` и `DecreaseHours(int)`. Содержит собственные операторы ввода/вывода.

```cpp
class Teacher : public Person {
    string subject;
    int hours;
public:
    Teacher();
    ~Teacher();
    void Show();
    Teacher(string, int, string, int);
    Teacher(const Teacher&);
    string GetSubject();
    int GetHours();
    void SetSubject(string);
    void SetHours(int);
    void IncreaseHours(int);
    void DecreaseHours(int);
    Teacher& operator=(const Teacher&);
    friend istream& operator>>(istream&, Teacher&);
    friend ostream& operator<<(ostream&, const Teacher&);
};
```

#### 2.4. Класс `Vector`
Содержит динамический массив указателей на `Object`. Позволяет добавлять объекты через метод `Add(Object*)`. Перегруженная операция `<<` проходит по массиву и вызывает для каждого элемента виртуальный метод `Show()`, обеспечивая полиморфное поведение.

```cpp
class Vector {
    Object** beg;
    int size;
    int cur;
public:
    Vector();
    Vector(int);
    ~Vector();
    void Add(Object*);
    friend ostream& operator<<(ostream&, const Vector&);
};
```

### 3. Определение компонентных функций

#### 3.1. Класс `Object`
Конструктор и деструктор оставлены с пустыми телами (реализация в `.cpp`).  
Чисто виртуальная функция `Show()` не имеет реализации в базовом классе.

#### 3.2. Класс `Person`
- **Конструктор по умолчанию** инициализирует поля пустой строкой и нулём.
- **Переопределённый `Show()`** выводит имя и возраст.
- **Сеттеры/геттеры** обеспечивают доступ к полям.
- **Оператор присваивания** копирует значения полей с проверкой на самоприсваивание.
- **Деструктор** объявлен виртуальным для безопасного удаления в иерархии.

#### 3.3. Класс `Teacher`
- **Конструктор по умолчанию** вызывает конструктор `Person()` и обнуляет часы, очищает предмет.
- **Переопределённый `Show()`** выводит имя, возраст, предмет и часы.
- **Методы `IncreaseHours(int n)` и `DecreaseHours(int n)`** изменяют количество часов (уменьшение не допускает отрицательных значений).
- **Оператор присваивания** вызывает присваивание базовой части и копирует собственные поля.

#### 3.4. Класс `Vector`
- **Конструктор по умолчанию** обнуляет указатели.
- **Конструктор с параметром** выделяет массив из `n` указателей.
- **Деструктор** освобождает память массива (сами объекты не удаляются, так как хранятся вне вектора).
- **Метод `Add(Object* p)`** добавляет указатель в массив, если есть место.
- **Дружественный `operator<<`** выводит все элементы вектора, вызывая `Show()` через указатели.

### 4. Определение глобальных функций

#### 4.1. Операторы ввода/вывода для `Person`
```cpp
istream& operator>>(istream& in, Person& p) {
    cout << "Enter name: "; in >> p.name;
    cout << "Enter age: "; in >> p.age;
    return in;
}

ostream& operator<<(ostream& out, const Person& p) {
    out << "Name: " << p.name << ", Age: " << p.age;
    return out;
}
```

#### 4.2. Операторы ввода/вывода для `Teacher`
Аналогичны, но добавляют запрос и вывод полей `subject` и `hours`.

#### 4.3. Оператор вывода для `Vector`
```cpp
ostream& operator<<(ostream& out, const Vector& v) {
    if (v.size == 0) out << "Empty" << endl;
    Object** p = v.beg;
    for (int i = 0; i < v.cur; i++) {
        (*p)->Show(); // полиморфный вызов
        p++;
    }
    return out;
}
```
Здесь проявляется механизм позднего связывания: вызывается `Show()` того класса, на объект которого реально указывает указатель.

### 5. Функция main()

```cpp
int main() {
    Vector v(5);          // вектор из 5 элементов
    Person p;
    Teacher t;

    cout << "Enter data for Person:" << endl;
    cin >> p;
    cout << "Enter data for Teacher:" << endl;
    cin >> t;

    Object* ptr = &p;
    v.Add(ptr);           // добавить Person
    ptr = &t;
    v.Add(ptr);           // добавить Teacher

    cout << "\nVector contents (polymorphic Show):" << endl;
    cout << v;            // вывод через operator<< вектора

    // Демонстрация методов Teacher
    cout << "\n--- Change teacher hours ---" << endl;
    t.IncreaseHours(10);
    cout << "After increase: " << t << endl;
    t.DecreaseHours(5);
    cout << "After decrease: " << t << endl;

    system("pause");
    return 0;
}
```

Программа запрашивает данные для `Person` и `Teacher`, добавляет их адреса в вектор, а затем выводит вектор, вызывая для каждого объекта его собственную версию `Show()`.

### 7. Ответы на контрольные вопросы

1. **Какой метод называется чисто виртуальным? Чем он отличается от виртуального метода?**  
   Чисто виртуальный метод объявляется с инициализатором `= 0` и не имеет тела в базовом классе. Обычный виртуальный метод имеет реализацию, которая может быть переопределена в производных классах. Чисто виртуальный метод обязан быть переопределён в производном классе (если только класс не остаётся абстрактным).

2. **Какой класс называется абстрактным?**  
   Класс, содержащий хотя бы один чисто виртуальный метод. Объекты такого класса создать нельзя, он используется только как базовый для иерархии.

3. **Для чего предназначены абстрактные классы?**  
   Для представления общих понятий, которые предполагается конкретизировать в производных классах. Они задают интерфейс, который должны реализовать наследники, и обеспечивают полиморфное поведение.

4. **Что такое полиморфные функции?**  
   Функции, способные работать с объектами разных типов, принадлежащих одной иерархии. Обычно принимают указатель или ссылку на базовый класс и вызывают виртуальные методы, реальное поведение которых определяется динамическим типом объекта.

5. **Чем полиморфизм отличается от принципа подстановки?**  
   Принцип подстановки (Барбары Лисков) говорит о том, что объект производного класса может использоваться везде, где ожидается объект базового, не нарушая корректности программы. Полиморфизм же обеспечивает вызов методов в соответствии с фактическим типом объекта во время выполнения, что является механизмом реализации этого принципа.

6. **Привести примеры иерархий с использованием абстрактных классов.**  
   - `Shape` (абстрактный) → `Circle`, `Rectangle`, `Triangle` (каждый реализует `area()`).  
   - `Animal` (абстрактный, `sound()=0`) → `Dog`, `Cat` (конкретные звуки).  
   - `Vehicle` (абстрактный) → `Car`, `Truck` (разная реализация `move()`).  

7. **Привести примеры полиморфных функций.**  
   ```cpp
   void print(Object& obj) { obj.Show(); }
   ```
   Или функция, сортирующая массив указателей на `Comparable` (абстрактный класс с чисто виртуальной `compare()`).  
   В данной лабораторной работе полиморфной является операция вывода вектора `operator<<`, которая вызывает `Show()` у каждого элемента.

8. **В каких случаях используется механизм позднего связывания?**  
   Когда вызов метода осуществляется через указатель или ссылку на базовый класс, а фактический объект может быть производного типа. Решение о том, какая версия метода будет вызвана, принимается во время выполнения на основе динамического типа объекта. Это ключевая техника для реализации полиморфизма.

## 2. Описание класса

Класс `Number` представляет составное число, образованное целой и дробной частями.

```cpp
class Number {
public:
    unsigned int first;   // положительная целая часть
    double second;        // положительная дробная часть

    Number();                          // конструктор по умолчанию
    void Init(int f, double s);        // инициализация полей с проверкой
    void Read();                       // ввод с клавиатуры
    void Show() const;                 // вывод на экран
    double multiply(double k);         // умножение значения на число k
    double getValue() const;           // получение значения first + second
};
```


## 3. Определение компонентных функций
1. Конструктор по умолчанию
```cpp
Number::Number() {
    first = 1;
    second = 0.0;
}
Устанавливает начальные значения: целая часть = 1, дробная = 0.0.

Метод Init (модификатор)
cpp
void Number::Init(int f, double s) {
    if (f <= 0) {
        cout << "Ошибка: целая часть должна быть > 0" << endl;
        f = 1;
    }
    if (s < 0) {
        cout << "Ошибка: дробная часть должна быть >= 0" << endl;
        s = 0.0;
    }
    first = f;
    second = s;
}
```
Контролирует корректность входных значений. При нарушении выводит сообщение и присваивает безопасные значения, после чего инициализирует поля.

2. Метод Read
```cpp
void Number::Read() {
    int f;
    double s;
    cout << "Введите целую часть (>0): ";
    cin >> f;
    cout << "Введите дробную часть (>=0): ";
    cin >> s;
    Init(f, s);   // передача в Init для проверки
}
Запрашивает у пользователя значения и сразу обеспечивает проверку через Init.
```

3. Метод Show

void Number::Show() const {

## 4. Определение внешней функции make_Number

```cpp
Number make_Number(int f, double s) {
    Number temp;
    temp.Init(f, s);
    return temp;
}
```
Функция создает временный объект Number, инициализирует его переданными значениями (с проверкой через Init) и возвращает готовый объект. Позволяет создавать инициализированный экземпляр одной строкой:

```cpp
Number num = make_Number(3, 0.14);
```

Инициализация Init – проверка корректности значений (отрицательные или нулевые значения отвергаются с выдачей предупреждения).

## 6. Ответы на контрольные вопросы
1. Что такое класс?
> Класс – абстрактный тип данных, определяемый пользователем, модель объекта, объединяющая данные (поля) и функции для работы с ними (методы).

2. Что такое объект (экземпляр) класса?
> Объект – конкретная переменная типа класса. Класс описывает структуру, а объекты – её реальные представители в памяти программы.

3. Как называются поля класса?
>Поля класса также называют атрибутами или данными-членами.

4. Как называются функции класса?
>Функции класса называются методами.

5. Для чего используются спецификаторы доступа?
>Спецификаторы доступа (private, public, protected) управляют видимостью членов класса для внешнего кода и производных классов, обеспечивая инкапсуляцию.

6. Для чего используется спецификатор public?
>Члены, объявленные как public, доступны из любой части программы, где виден объект класса. Обычно в public помещают интерфейс класса.

7. Для чего используется спецификатор private?
>Члены private доступны только внутри методов самого класса. Это скрывает внутреннюю реализацию и предотвращает случайное изменение данных извне.

8. Если описание класса начинается со слова class, то какой спецификатор доступа используется по умолчанию?
>Для class по умолчанию действует private.

9. Если описание класса начинается со слова struct, то какой спецификатор доступа используется по умолчанию?
> Для struct по умолчанию действует public.

10. Какой спецификатор доступа должен использоваться при описании интерфейса класса? Почему?
> Интерфейс класса должен быть описан в секции public, чтобы им могли пользоваться внешние функции и другие классы. Закрытый (private) интерфейс не позволяет обращаться к методам и полям.

11. Каким образом можно изменить значения атрибутов экземпляра класса?
> Если атрибут в public, его можно изменить напрямую: object.attr = value;. Если он в private, то только через специальные методы-модификаторы (сеттеры).

12. Каким образом можно получить значения атрибутов экземпляра класса?
>Для public-атрибутов – напрямую. Для private – через методы-селекторы (геттеры), возвращающие значения.

13. Обращение к полю name объекта s (описан как Student *s = new Student; в struct Student):
> s->name (так как struct даёт public доступ).

14. Обращение к полю name объекта s (описан как Student s; в struct Student):
> s.name

15. Обращение к полю name объекта s (описан как Student *s = new Student; в class Student с private по умолчанию):
> Напрямую обратиться нельзя, так как поле name по умолчанию private. Если не предоставлен публичный метод доступа, компилятор выдаст ошибку.

16. Обращение к полю name объекта s (описан как Student s; в class Student с public: char* name;):
> s.name (поле открыто).

17. Обращение к полю name объекта s (описан как Student *s = new Student; в class Student с public: char* name;):
> s->name

#### 3.1. Задача 1 (глобальные функции для `multimap<float>`)

Все функции работают с типом `multimap<int, float>`, ключ – целое число, значение – `float`.

**Функция поиска минимального элемента**
```cpp
CIt find_min(const MMap& m) {
    if (m.empty()) return m.end();
    CIt minIt = m.begin();
    for (CIt it = m.begin(); it != m.end(); ++it)
        if (it->second < minIt->second) minIt = it;
    return minIt;
}
```

**Функция поиска максимального элемента**
```cpp
CIt find_max(const MMap& m) {
    if (m.empty()) return m.end();
    CIt maxIt = m.begin();
    for (CIt it = m.begin(); it != m.end(); ++it)
        if (it->second > maxIt->second) maxIt = it;
    return maxIt;
}
```

**Функция вычисления среднего арифметического**
```cpp
float average(const MMap& m) {
    if (m.empty()) return 0;
    float sum = 0;
    for (CIt it = m.begin(); it != m.end(); ++it)
        sum += it->second;
    return sum / m.size();
}
```

**Функция добавления минимального элемента по заданному ключу (задание 3)**
```cpp
void add_min_at_key(MMap& m, int key) {
    CIt minIt = find_min(m);
    if (minIt != m.end()) {
        float minVal = minIt->second;
        m.insert(make_pair(key, minVal));
        cout << "Added min " << minVal << " at key " << key << endl;
    }
}
```

**Функция удаления элементов, меньших среднего арифметического (задание 4)**
```cpp
void remove_less_than_average(MMap& m) {
    float avg = average(m);
    for (It it = m.begin(); it != m.end(); ) {
        if (it->second < avg)
            it = m.erase(it);
        else
            ++it;
    }
}
```

**Функция деления всех элементов на максимальный (задание 5)**
```cpp
void divide_by_max(MMap& m) {
    if (m.empty()) return;
    CIt maxIt = find_max(m);
    float maxVal = maxIt->second;
    if (maxVal == 0) {
        cout << "Max is zero, division impossible." << endl;
        return;
    }
    for (It it = m.begin(); it != m.end(); ++it)
        it->second /= maxVal;
}
```

### 5. Объяснение результатов работы программы

Программы корректно реализуют все три части задания.

- Контейнер `multimap` обеспечивает хранение пар ключ-значение с возможностью дублирования ключей. Ключи целочисленные, что имитирует работу со списком.
- Поиск минимума/максимума выполняется полным перебором значений через итераторы.
- Среднее арифметическое вычисляется путём суммирования всех значений и деления на количество элементов. Для пользовательского типа `Pair` используются перегруженные операторы `+` и `/`.
- Добавление минимального элемента по заданному ключу осуществляется вставкой новой пары `make_pair(key, min)`; если такой ключ уже существует, multimap сохраняет оба элемента.
- Удаление элементов, меньших среднего, производится безопасным обходом с постфиксным инкрементом итератора, так как `erase` инвалидирует текущий итератор.
- Деление на максимум использует ссылки на значения, модифицируя их непосредственно. Для `float` это обычное деление, для `Pair` — оператор деления пары на пару.
- Параметризированный класс `MyList` демонстрирует обобщённый подход: один и тот же код работает с любым типом `T`, для которого определены необходимые операции (`+`, `/`, `<`).

Все операции отработаны последовательно, вывод контейнера после каждого шага подтверждает корректность изменений.

### 6. Ответы на контрольные вопросы

1. **Что представляет собой ассоциативный контейнер?**  
   Ассоциативный контейнер — это структура данных, хранящая пары «ключ – значение» и обеспечивающая быстрый доступ к значению по ключу. Элементы автоматически упорядочиваются по ключу.

2. **Перечислить ассоциативные контейнеры библиотеки STL.**  
   `map`, `multimap`, `set`, `multiset`.

3. **Каким образом можно получить доступ к элементам ассоциативного контейнера?**  
   С помощью итераторов (прямых и обратных), операции индексации `[]` (только для `map`), методов `find()`, `lower_bound()`, `upper_bound()`, `count()`.

4. **Привести примеры методов, используемых в ассоциативных контейнерах.**  
   `insert()`, `erase()`, `clear()`, `swap()`, `empty()`, `size()`, `find()`, `count()`, `lower_bound()`, `upper_bound()`, `key_comp()`, `value_comp()`.

5. **Каким образом можно создать контейнер map? Привести примеры.**  
   - `map<int, string> m1;` – пустой словарь.  
   - `map<int, string> m2(m1);` – копирование.  
   - `map<int, string> m3(begin, end);` – из диапазона итераторов.

6. **Каким образом упорядочены элементы в контейнере map по умолчанию? Как изменить порядок на обратный?**  
   По умолчанию — по возрастанию ключа (с помощью `less<Key>`). Для обратного порядка нужно указать `greater<Key>` в качестве третьего параметра шаблона: `map<int, string, greater<int>>`.

7. **Какие операции определены для контейнера map?**  
   Все операции, общие для контейнеров STL: вставка, удаление, поиск, доступ по ключу `[]`, сравнение (`==`, `!=`, `<`, `>`, `<=`, `>=`), присваивание.

8. **Написать функцию для добавления элементов в контейнер map с помощью функции make_pair().**  
   ```cpp
   void add_pair(map<int, string>& m, int key, const string& val) {
       m.insert(make_pair(key, val));
   }
   ```

9. **Написать функцию для добавления элементов в контейнер map с помощью операции прямого доступа [].**  
   ```cpp
   void add_by_index(map<int, string>& m, int key, const string& val) {
       m[key] = val;
   }
   ```

10. **Написать функцию для печати контейнера map с помощью итератора.**  
    ```cpp
    void print_map(const map<int, string>& m) {
        for (auto it = m.begin(); it != m.end(); ++it)
            cout << it->first << " : " << it->second << endl;
    }
    ```

11. **Написать функцию для печати контейнера map с помощью операции прямого доступа [].**  
    ```cpp
    void print_by_index(const map<int, string>& m) {
        for (const auto& p : m) // или по i от 0 до size() с [i] – для map с последовательными ключами

    }
    ```
    (Примечание: `[]` требует существования элемента, поэтому для обхода надёжнее использовать range‑based loop.)

12. **Чем отличаются контейнеры map и multimap?**  
    `map` не допускает дублирования ключей (каждый ключ уникален), поддерживает оператор `[]`.  
    `multimap` разрешает хранение нескольких элементов с одинаковым ключом, оператор `[]` отсутствует.

13. **Что представляет собой контейнер set?**  
    `set` — ассоциативный контейнер, хранящий только ключи (без значений). Все ключи уникальны и отсортированы.

14. **Чем отличаются контейнеры map и set?**  
    `map` хранит пары ключ-значение, позволяя по ключу извлекать значение.  
    `set` хранит только ключи; значение и ключ — одно и то же.

15. **Каким образом можно создать контейнер set? Привести примеры.**  
    - `set<int> s1;` – пустое множество.  
    - `int arr[] = {1,2,3}; set<int> s2(arr, arr+3);` – из массива.  
    - `set<int> s3(s2);` – копирование.

16. **Каким образом упорядочены элементы в контейнере set по умолчанию? Как изменить порядок на обратный?**  
    По умолчанию по возрастанию (`less<T>`). Чтобы изменить на обратный, задают `greater<T>` вторым параметром: `set<int, greater<int>>`.

17. **Какие операции определены для контейнера set?**  
    Вставка `insert()`, удаление `erase()`, поиск `find()`, `count()`, `lower_bound()`, `upper_bound()`, а также все стандартные операции контейнеров (размер, очистка и т.д.).

18. **Написать функцию для добавления элементов в контейнер set.**  
    ```cpp
    void add_to_set(set<int>& s, int value) {
        s.insert(value);
    }
    ```
